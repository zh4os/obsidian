# 提示词链

又称管道模式（Pipeline pattern）。将复杂任务分解为一系列较小的、相互关联的子任务，每个子任务通过专门设计的提示词单独处理，前一个提示词的输出作为下一个提示词的输入，逐步构建最终解决方案。

## 核心思想

- **分而治之**：复杂任务在单一提示词内处理时，LLM 容易出现指令忽略、上下文偏离、错误传播、幻觉等问题
- **模块化**：每个步骤可以精心设计和优化，专注于问题的特定方面
- **可调试**：链中的每个环节都可以独立检查和修复
- **结构化输出**：步骤间传递 JSON/XML 等结构化数据，减少歧义

## 应用场景

- **信息处理工作流**：文档提取 → 总结 → 实体抽取 → 知识库搜索 → 报告生成
- **复杂查询回答**：子问题拆解 → 分别检索 → 综合答案
- **数据提取与转换**：从非结构化来源（发票、表单）提取 → 验证 → 结构化输出
- **内容生成**：构思 → 大纲 → 分段起草 → 审查完善
- **对话智能体**：每轮对话构建新提示词，融合历史上下文
- **代码生成与完善**：伪代码 → 初始代码 → 静态分析 → 重写 → 测试
- **多模态推理**：文本提取 → 标签关联 → 表格解读

## 上下文工程

提示词链是上下文工程（Context Engineering）的实践基础。上下文工程不仅优化提示词措辞，更系统地构建 AI 模型的完整信息环境——包括系统提示词、检索文档、工具输出、用户身份、交互历史等多层信息。

## 实践建议

> 当任务对于单个提示词过于复杂、涉及多个不同的处理阶段、需要在步骤之间与外部工具交互，或者在构建需要执行多步推理并维护状态的智能体系统时，使用此模式。
