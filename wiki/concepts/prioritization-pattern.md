# 优先级排序

任务优先级判断和资源分配策略。当智能体面对多个待处理任务时，需要根据紧急性、重要性、依赖关系和资源约束，决定先做什么、后做什么、以及分配多少资源。

## 排序维度

- **紧急性**：任务的时间敏感度
- **重要性**：任务对最终目标的影响权重
- **依赖关系**：任务之间的前后依赖
- **资源需求**：任务所需的计算资源和时间
- **用户期望**：用户对不同任务的优先级偏好

## 应用场景

- **多任务处理**：同时收到多个用户请求时的处理顺序
- **资源受限环境**：在有限 API 配额下优先处理最重要的任务
- **工作流编排**：复杂工作流中子任务的执行顺序优化
- **中断管理**：新任务到来时是否打断当前任务

## 实践建议

> 当 agent 需要在多个竞争任务中做出取舍时，使用优先级排序模式。结合紧急性和重要性两个维度评估。
