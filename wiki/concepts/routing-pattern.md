# 路由

将条件逻辑引入智能体的操作框架，使其从固定的执行路径转变为动态决策模型：智能体动态评估特定标准，从一组可能的后续操作中进行选择。路由使系统行为更加灵活且具有上下文感知能力。

## 路由实现方式

| 方式 | 说明 | 特点 |
|---|---|---|
| **基于 LLM 的路由** | 提示 LLM 分析输入并输出分类标识 | 灵活，能处理复杂语义 |
| **基于嵌入的路由** | 将输入转为向量，与路由嵌入比较相似度 | 基于语义而非关键词 |
| **基于规则的路由** | if-else、switch case、关键词匹配 | 快速、确定性强，灵活性较差 |
| **基于 ML 模型的路由** | 在标记数据上训练分类器 | 专门化，推理时无需 LLM |

## 应用场景

- **客户服务**：根据用户意图路由到不同专业智能体（订单查询、技术支持、账户管理）
- **数据处理管道**：按内容/格式分类，定向到不同处理流程
- **研究系统**：根据任务类型分配到搜索、总结、分析等不同智能体
- **AI 编程助手**：识别用户意图（调试、解释、翻译），路由到专门工具

## 实践建议

> 当智能体必须根据用户输入或当前状态在多个不同的工作流、工具或子智能体之间做出决策时，使用路由模式。

## 代码框架

书中示例使用 **LangChain / LangGraph** 和 **Google ADK**。LangGraph 基于图结构提供可视化和显式的状态转换；Google ADK 通过定义工具和子智能体实现委托路由。
